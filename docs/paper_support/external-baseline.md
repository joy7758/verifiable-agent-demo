# External Baseline

The external baseline adds a deterministic `external_baseline` mode under `artifacts/runs/<task_id>/external_baseline/`.

Design choice:

- The baseline is a thin local adapter shaped after the repository's existing CrewAI demo surface.
- It preserves CrewAI-like concepts such as agent initialization, crew task dispatch, callback logging, and task completion.
- It does not claim live end-to-end CrewAI governance, integrity, or receipt support.

What is real vs shimmed:

- Real alignment: naming, layering, and output shape are grounded in the repository's existing `crew/crew_demo.py`.
- Shimmed: the per-task export is generated by the local paper harness, not by running a live CrewAI runtime 16 times.
- Honest omission: no explicit intent object, no governance checkpoint, no integrity proof, and no bounded receipt are claimed for this mode.

Why it matters:

- It gives the paper a more realistic "default framework observability surface" than the local trace-only baseline.
- It remains deterministic and cheap to reproduce.

Run command:

```bash
make eval-external-baseline
```
